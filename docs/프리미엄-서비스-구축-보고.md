### 2.2 스마트 이상 감지 (6h/9h/12h)

| 단계 | 무활동 시간 | 알림 문구 (이동 없음/안전 확인 톤) |
|------|-------------|----------------------------------|
| 1 | 6시간 | 6시간째 움직임이 없습니다. 안전 확인이 필요합니다. |
| 2 | 9시간 | 9시간째 움직임이 없습니다. 즉시 확인해 주세요. |
| 3 | 12시간 | 12시간째 움직임이 없습니다. 긴급 확인이 필요합니다. |

**기준 시각**

- **원칙**: `lastMovementAt` (핸드폰 물리적 이동, Activity Recognition)
- **보조**: `lastMovementAt`이 없거나 **센서 비정상(24h 이상 오래됨)** 시에만 `lastResponseAt`(앱 기록) 사용
- **센서 비정상 상수**: `MOVEMENT_STALE_HOURS = 24` (코드 고정)

**Cloud Function**

- `sendPremiumInactivityAlert` — 30분마다 실행 (`*/30 * * * *`, Asia/Seoul)
- `lastMovementAt` 원칙, 없을 때만 `lastResponseAt` 보조로 단계 판별 → 프리미엄 보호자에게만 FCM 발송

**안전장치**

- **(A) 같은 단계 재발송 방지**: 동일 단계에서 3시간 이내 재발송 스킵
- **(B) 단계 역행 방지**: `currentStage <= lastInactivityAlertLevel` 시 스킵

### 2.3 실시간 무활동 알림 (6h/9h/12h)

| 구분 | 내용 |
|------|------|
| **발송 시점** | 6h/9h/12h 도달 시 (30분 주기 스캔) |
| **수신자** | 해당 보호대상자와 연결된 **프리미엄 보호자만** (무료 보호자 절대 발송 금지) |
| **FCM data.type** | `INACTIVITY_ALERT` |

### 2.4 3일 무응답 알림 (72h)

| 구분 | 내용 |
|------|------|
| **발송 시점** | 매일 20:05, lastResponseAt < 72h |
| **수신자** | 해당 보호대상자와 연결된 모든 보호자 |
| **FCM data.type** | `NO_RESPONSE_ALERT` |
| **톤** | 루틴/관계 알림 (부담 낮게) |
| **중복 방지** | `lastNoResponseAlertAt` (72h 이내 재발송 스킵) |
| **리셋** | onResponseCreated/Updated에서 lastNoResponseAlertAt delete |

**Flutter: 핸드폰 이동 감지**

- `MovementDetectionService` — Activity Recognition (WALKING, RUNNING 등) 감지 시 `lastMovementAt` 업데이트
- **앱 부팅/로그인 시 상시 유지** — `main.dart` `_AppLifecycleHandler`에서 AuthService 리스너로 start/stop (SubjectModeScreen 진입 여부와 무관)
- **10분 health check** — 서비스 미동작 시 자동 재시작 (기기 재부팅·강제 종료 대응)
- **배터리 최적화 제외 가이드** — 설정 화면에 안내·버튼 노출 (Android)
- 15분 debounce — Firestore write·배터리·비용 절감
- `runForegroundService: true` — 배경에서도 감지 (Android)

**Flutter 라우팅**

- `fcm_service.dart` — INACTIVITY_ALERT 타입 처리
- `splash_screen.dart` — 알림 클릭 시 보호자 대시보드 진입
- `notification_service.dart` — 알림 페이로드 처리

## 3. 데이터 구조

### 3.1 users (구독·등록 수)

| 필드 | 타입 | 용도 |
|------|------|------|
| `subscriptionStatus` | string | `active` / `premium` → 프리미엄 |
| `subscriptionExpiry` | Timestamp | 구독 만료일 |
| `guardianSubjectCount` | number | 보호자별 등록 대상자 수 |

### 3.2 subjects (무활동·무응답 알림용)

| 필드 | 타입 | 용도 |
|------|------|------|
| `lastMovementAt` | Timestamp \| null | 마지막 핸드폰 물리적 이동 시각 (Activity Recognition) |
| `lastInactivityAlertAt` | Timestamp \| null | 마지막 무이동(6/9/12h) 알림 발송 시각 |
| `lastInactivityAlertLevel` | number | 마지막 무이동 알림 단계 (0=미발송, 1=6h, 2=9h, 3=12h) |
| `lastNoResponseAlertAt` | Timestamp \| null | 마지막 3일 무응답(72h) 알림 발송 시각 (중복 방지 전용) |

**lastMovementAt 갱신**

- `MovementDetectionService`: WALKING, RUNNING 등 감지 시 클라이언트가 Firestore 업데이트 (15분 debounce)
- `onResponseCreated`/`onResponseUpdated`: 안부 기록 시 서버가 갱신 (센서 미동작 시 보조)

**리셋 조건**

- **무이동(6/9/12h)**: onResponseCreated/Updated, onSubjectUpdated에서 lastInactivityAlert* 초기화
- **무응답(72h)**: onResponseCreated/Updated에서 lastNoResponseAlertAt delete (무이동 리셋과 분리)
- **lastMovementAt**: onResponseCreated/Updated에서 갱신 (센서 보조)

## 4. 보안·규칙

### 4.1 Firestore Rules

- `lastInactivityAlertAt`, `lastInactivityAlertLevel`, `lastNoResponseAlertAt` — **클라이언트 쓰기 차단** (서버 전용)
- `subscriptionStatus`, `subscriptionExpiry`, `subscriptionUpdatedAt` — **클라이언트 쓰기 차단** (서버 전용)
- `pairedGuardianUids` — 보호자 추가/제거는 Callable Function 전용

### 4.2 Cloud Functions

- `isPremium(userData)` — subscriptionStatus + subscriptionExpiry 검증
- `FREE_GUARDIAN_SUBJECT_LIMIT = 2` — 무료 플랜 제한

## 5. Cloud Functions 목록 (프리미엄 관련)

| 함수 | 스케줄/트리거 | 역할 |
|------|--------------|------|
| `addGuardianToSubject` | Callable | 보호자→대상자 추가 (프리미엄 체크) |
| `addGuardianToSubjectBySubjectInvite` | Callable | 대상자→보호자 추가 (프리미엄 체크) |
| `removeGuardianFromSubject` | Callable | 보호자 제거 (guardianSubjectCount -1) |
| `removeGuardianFromSubjectBySubject` | Callable | 대상자→보호자 제거 (guardianSubjectCount -1) |
| `processPendingInvitesOnSignup` | Auth onCreate | 신규 가입 시 대기 초대 처리 (프리미엄 체크) |
| `onResponseCreated` | Firestore onCreate | 기록 시 보호자 알림 + lastInactivityAlert* 리셋 + lastMovementAt 갱신 |
| `onResponseUpdated` | Firestore onUpdate | 기록 수정 시 보호자 알림 + lastInactivityAlert* 리셋 + lastMovementAt 갱신 |
| `onSubjectUpdated` | Firestore onUpdate | lastMovementAt 갱신 시 lastInactivityAlert* 리셋 (루프 방지: lastInactivityAlertLevel 이미 0이면 update 스킵) |
| `sendPremiumInactivityAlert` | Pub/Sub 30분마다 | 6h/9h/12h 무이동 알림 (프리미엄 보호자만, 센서 비정상 24h) |
| `sendThreeDayNoResponseAlert` | Pub/Sub 20:05 매일 | 72h 무응답 알림 (lastNoResponseAlertAt 중복 방지) |
| `verifyPlaySubscription` | Callable | Google Play 구독 검증 → subscriptionStatus/Expiry 갱신 |
| `syncPremiumExpiry` | Pub/Sub 00:10 매일 | 만료된 premium 사용자 inactive 처리 |
