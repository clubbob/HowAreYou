# 테스트 전화번호 설정 가이드

## 개요
두 개의 폰(Pixel 6, Pixel 7)을 사용하여 보호자와 보호 대상자 기능을 테스트합니다.

## 테스트 전화번호

### 1. 보호자 (Pixel 6)
- **전화번호**: `010-1111-2222`
- **E.164 형식**: `+821011112222`
- **인증 코드**: `111111`
- **역할**: 보호자 모드로 사용

### 2. 보호 대상자 (Pixel 7)
- **전화번호**: `010-3333-4444`
- **E.164 형식**: `+821033334444`
- **인증 코드**: `333333`
- **역할**: 보호 대상자 모드로 사용

## Firebase Console 설정 방법

### 1. Firebase Console 접속
1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 프로젝트 선택

### 2. Authentication 설정
1. 왼쪽 메뉴에서 **Authentication** 클릭
2. **Sign-in method** 탭 클릭
3. **전화번호** (Phone) 클릭
4. **사용 설정** 토글이 켜져 있는지 확인

### 3. 테스트 전화번호 추가
1. **테스트 전화번호** 섹션으로 스크롤
2. 기존 테스트 번호가 있다면 모두 삭제
3. 다음 두 개의 테스트 번호 추가:

#### 보호자 번호 추가
- **전화번호**: `+82 10-1111-2222` 또는 `+821011112222`
- **인증 코드**: `111111`
- **추가** 버튼 클릭

#### 보호 대상자 번호 추가
- **전화번호**: `+82 10-3333-4444` 또는 `+821033334444`
- **인증 코드**: `333333`
- **추가** 버튼 클릭

### 4. 설정 확인
- 두 개의 테스트 번호가 목록에 표시되는지 확인
- 각 번호의 인증 코드가 올바르게 설정되었는지 확인

## 테스트 절차

### 1단계: 보호 대상자 (Pixel 7) 설정
1. Pixel 7에서 앱 실행
2. 전화번호 입력: `010-3333-4444`
3. "인증 코드 전송" 클릭
4. 인증 코드 입력: `333333`
5. "인증 확인" 클릭
6. **보호 대상자 모드** 선택
7. 보호자 지정 (다음 단계에서 추가할 보호자 번호 입력)

### 2단계: 보호자 (Pixel 6) 설정
1. Pixel 6에서 앱 실행
2. 전화번호 입력: `010-1111-2222`
3. "인증 코드 전송" 클릭
4. 인증 코드 입력: `111111`
5. "인증 확인" 클릭
6. **보호자 모드** 선택
7. 보호 대상자 확인 대시보드에서 상태 확인

### 3단계: 기능 테스트
1. **보호 대상자 (Pixel 7)**:
   - 아침/점심/저녁 시간에 질문 화면 표시 확인
   - 기분 선택 (좋아/보통/안좋아)
   - 상태 저장 확인

2. **보호자 (Pixel 6)**:
   - 보호 대상자 대시보드에서 상태 확인
   - 실시간 알림 수신 확인
   - 미회신 알림 확인

## 주의사항

- 테스트 전화번호는 개발 중에만 사용합니다
- 실제 SMS는 전송되지 않습니다
- Firebase Console에서 설정한 인증 코드를 정확히 입력해야 합니다
- 전화번호는 `010-1111-2222` 형식으로 입력해도 자동으로 `+821011112222`로 변환됩니다

## 문제 해결

### 인증 코드가 작동하지 않는 경우
1. Firebase Console에서 테스트 번호가 올바르게 등록되었는지 확인
2. 전화번호 형식이 `+821011112222` (E.164)인지 확인
3. 인증 코드가 정확히 일치하는지 확인 (공백 없이)

### 보호자/보호 대상자 연결이 안 되는 경우
1. 두 번호 모두 앱에 로그인되어 있는지 확인
2. 보호 대상자가 보호자를 지정했는지 확인
3. Firestore 데이터베이스에서 `subjects` 컬렉션 확인
