# Firebase Console 현재 화면 - 다음 단계

## 현재 상태 확인

### ✅ 확인된 사항:
- **전화번호 인증**: "사용 설정됨" (활성화됨) ✅
- **일일 SMS 할당량**: 1,000건 (테스트 번호 사용 시 영향 없음)

## 다음 단계: 테스트 전화번호 확인

### 1단계: 전화번호 카드 클릭

현재 화면에서 **"전화" (Phone)** 카드를 클릭하세요.

### 2단계: 테스트 전화번호 섹션으로 이동

전화번호 설정 화면이 열리면:
1. **"테스트 전화번호"** 섹션으로 스크롤
2. 다음 번호들이 등록되어 있는지 확인:

#### 등록되어 있어야 할 번호:

##### 보호자 번호 (Pixel 1)
- **전화번호**: `+821011112222` 또는 `+82 10-1111-2222`
- **인증 코드**: `111111`

##### 보호 대상자 번호 (Pixel 2)
- **전화번호**: `+821033334444` 또는 `+82 10-3333-4444`
- **인증 코드**: `333333`

## 확인 체크리스트

- [ ] 전화번호 카드를 클릭했는가?
- [ ] 테스트 전화번호 섹션이 보이는가?
- [ ] 보호자 번호 `+821011112222`가 등록되어 있는가?
- [ ] 보호자 인증 코드가 `111111`인가?
- [ ] 보호 대상자 번호 `+821033334444`가 등록되어 있는가?
- [ ] 보호 대상자 인증 코드가 `333333`인가?

## 테스트 번호가 없는 경우

### 추가 방법:

1. **"테스트 전화번호 추가"** 버튼 클릭
2. **전화번호 입력**: `+821011112222`
3. **인증 코드 입력**: `111111`
4. **추가** 버튼 클릭
5. 두 번째 번호도 동일하게 추가:
   - 전화번호: `+821033334444`
   - 인증 코드: `333333`
6. **저장** 확인

## 참고사항

### 일일 SMS 할당량 경고:
- 현재 화면에 표시된 "일일 1,000건 SMS 전송 할당량" 경고는 **테스트 번호 사용 시 영향 없습니다**
- 테스트 번호를 사용하면 실제 SMS가 전송되지 않으므로 할당량을 소비하지 않습니다
- 개발 중에는 테스트 번호만으로 충분합니다

### SMS 다중 인증(MFA):
- 현재 화면의 "SMS 다중 인증(MFA)" 섹션은 개발 단계에서는 필요 없습니다
- 프로덕션 배포 시 고려할 수 있습니다
