# 에뮬레이터 선택 실행 가이드

## 개요
여러 에뮬레이터가 실행 중일 때 원하는 에뮬레이터를 선택하여 앱을 실행할 수 있습니다.

## 사용 방법

### 방법 1: 선택 스크립트 사용 (권장)

#### PowerShell에서:
```powershell
.\run-select-device.ps1
```

#### CMD에서:
```cmd
run-select.bat
```

#### 실행 화면 예시:
```
연결된 기기 확인 중...

사용 가능한 에뮬레이터:

  [1] Pixel 6 (보호자) - emulator-5554
  [2] Pixel 7 (보호 대상자) - emulator-5556

사용할 에뮬레이터 번호를 선택하세요 (1-2): 
```

번호를 입력하면 해당 에뮬레이터에서 앱이 실행됩니다.

### 방법 2: 직접 기기 ID 지정

#### 특정 에뮬레이터 지정:
```powershell
.\run-android.ps1 -d emulator-5554
```

또는

```powershell
.\run-android.ps1 -d emulator-5556
```

### 방법 3: flutter run 직접 사용

```powershell
flutter run -d emulator-5554
```

**주의**: 이 방법은 EGL 로그가 그대로 출력됩니다.

## 에뮬레이터 구분하기

### 현재 실행 중인 에뮬레이터 확인:
```powershell
flutter devices
```

### 출력 예시:
```
Found 5 connected devices:
  sdk gphone64 x86 64 (mobile) • emulator-5554 • android-x64 • Android 14 (API 34) (emulator)
  sdk gphone64 x86 64 (mobile) • emulator-5556 • android-x64 • Android 14 (API 34) (emulator)
```

### 에뮬레이터 이름 변경 (Android Studio)

에뮬레이터를 더 쉽게 구분하려면 Android Studio에서 이름을 변경하세요:

1. **Android Studio** 실행
2. **Device Manager** 열기
3. 에뮬레이터 옆 **연필(편집)** 아이콘 클릭
4. **AVD Name** 변경:
   - Pixel 6: `Pixel_6_Guardian` 또는 `Pixel_6_보호자`
   - Pixel 7: `Pixel_7_Subject` 또는 `Pixel_7_보호대상자`
5. **Finish** 클릭

이름을 변경하면 `flutter devices` 출력에서도 구분이 쉬워집니다.

## 빠른 실행 가이드

### 보호자 (Pixel 6)에서 실행:
```powershell
.\run-android.ps1 -d emulator-5554
```

### 보호 대상자 (Pixel 7)에서 실행:
```powershell
.\run-android.ps1 -d emulator-5556
```

### 선택 스크립트 사용:
```powershell
.\run-select-device.ps1
```
→ 번호 선택하면 자동으로 실행

## 문제 해결

### 에뮬레이터가 목록에 안 보일 때
1. Android Studio에서 에뮬레이터가 실행 중인지 확인
2. `flutter devices` 명령어로 확인
3. 에뮬레이터 재시작

### 잘못된 기기 ID 오류
- `flutter devices`로 정확한 기기 ID 확인
- 기기 ID는 `emulator-`로 시작하는 형식
