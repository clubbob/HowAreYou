# 오늘 어때? 내부 테스트 안내

> Google Play 내부 테스트에 초대된 테스터를 위한 설치·연결 가이드

---

## 1. 연결 방식: **초대 링크**

이 앱은 **초대 링크**로 보호자와 보호대상자를 연결합니다.

- **보호자 → 보호대상자**: 보호자가 링크 생성 → 보호대상자에게 전송
- **보호대상자 → 보호자**: 보호대상자가 링크 생성 → 보호자에게 전송

링크를 받은 사람이 앱을 설치하고 로그인하면 **자동으로 연결**됩니다.

---

## 2. 상호 테스트가 되는 이유

| 역할 | 설치 | 연결 |
|------|------|------|
| **보호자(A)** | 내부 테스트 링크로 설치 | 앱에서 초대 링크 생성 → B에게 전송 |
| **보호대상자(B)** | A가 보낸 초대 링크 또는 내부 테스트 링크로 설치 | A의 링크로 들어와 로그인 → 자동 연결 |

둘 다 같은 내부 테스트 빌드를 쓰기 때문에, 알림·기록·상태 확인 등 **상호 기능이 그대로 동작**합니다.

---

## 3. 테스터 2인 1조 권장 시나리오

**A(보호자) ↔ B(보호대상자)** 한 쌍으로 테스트하는 것을 권장합니다.

### Step 1: 둘 다 앱 설치

1. 관리자가 보낸 **내부 테스트 참여 링크**를 각자 클릭
2. "테스터 되기" 수락
3. Play Store에서 **오늘 어때?** 설치

> ⚠️ **Gmail 계정 필요**: 테스터로 등록하려면 Google 계정( Gmail )이 필요합니다.

### Step 2: 한 명이 먼저 로그인

- **A(보호자)** 또는 **B(보호대상자)** 중 한 명이 먼저 앱을 열고 전화번호로 로그인

### Step 3: 초대 링크로 연결

**방법 A: 보호자 → 보호대상자**

1. A가 앱에서 **"보호 대상"** 역할 선택
2. **"보호 대상에게 초대 링크 보내기"** 탭
3. 카톡/문자로 B에게 링크 전송
4. B가 링크 클릭 → 앱 열림(또는 설치 후 열기) → **"시작하기"** → 로그인
5. 연결 완료

**방법 B: 보호대상자 → 보호자**

1. B가 앱에서 **"보호자"** 역할 선택
2. **"보호자에게 초대 링크 보내기"** 탭
3. 카톡/문자로 A에게 링크 전송
4. A가 링크 클릭 → 앱 열림 → **"시작하기"** → 로그인
5. 연결 완료

### Step 4: 상호 테스트 5가지

| # | 시나리오 | 확인 내용 |
|---|----------|-----------|
| 1 | **초대 수락** | 한쪽이 링크 보내고, 다른 쪽이 로그인 후 연결되는지 |
| 2 | **컨디션 기록** | 보호대상자가 오늘 안부 기록 → 보호자 화면에 반영되는지 |
| 3 | **보호자 알림** | 보호대상자가 기록하면 보호자에게 푸시 알림 오는지 |
| 4 | **19:00 리마인드** | 당일 미기록 시 저녁 7시에 보호대상자에게 알림 오는지 |
| 5 | **3일 무응답 알림** | 3일 연속 미기록 시 보호자에게 알림 오는지 |

---

## 4. 테스터에게 보낼 한 줄 안내

```
[오늘 어때?] 내부 테스트에 초대되었습니다.
1) 아래 링크로 "테스터 되기" 수락 후 앱 설치
2) 설치 후 앱에서 전화번호로 로그인
3) 상대방과 연결하려면: 앱 내 "초대 링크 보내기"로 링크를 보내세요. 받은 사람이 링크 클릭 후 로그인하면 자동 연결됩니다.

내부 테스트 참여: [Play Console에서 복사한 링크]
```

## 5. 자주 묻는 질문

**Q. 검색으로 앱을 찾을 수 있나요?**  
A. 아니요. 내부 테스트는 검색에 노출되지 않습니다. 반드시 **내부 테스트 참여 링크**로 접속해야 합니다.

**Q. Gmail이 없으면 되나요?**  
A. 테스터 등록에는 Google 계정이 필요합니다. Gmail이 아니어도 Google 계정이면 가능할 수 있으나, Gmail 사용을 권장합니다.

**Q. 초대 링크를 열었는데 앱이 안 열려요.**  
A. 먼저 내부 테스트 링크로 앱을 설치했는지 확인하세요. 앱이 없으면 초대 링크는 Play Store 설치 페이지로 이동합니다.
