# 개발 단계 테스트 번호 사용법

## 개요
개발 중에는 SHA-1 지문 등록 없이 테스트 전화번호만으로 인증을 테스트할 수 있습니다.

## 테스트 번호 설정

### Firebase Console에서 등록:

1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 프로젝트: **howareyou-1c5de**
3. **Authentication** → **Sign-in method** → **전화번호**
4. **테스트 전화번호** 섹션에서:

#### 보호자 (Pixel 1)
- 전화번호: `+821011112222`
- 인증 코드: `111111`

#### 보호 대상자 (Pixel 2)
- 전화번호: `+821033334444`
- 인증 코드: `333333`

## 사용 방법

### Pixel 1 (보호자)에서:
1. 앱 실행
2. 전화번호 입력: `010-1111-2222`
3. "인증 코드 전송" 클릭
4. 인증 코드 입력: `111111`
5. "인증 확인" 클릭

### Pixel 2 (보호 대상자)에서:
1. 앱 실행
2. 전화번호 입력: `010-3333-4444`
3. "인증 코드 전송" 클릭
4. 인증 코드 입력: `333333`
5. "인증 확인" 클릭

## 주의사항

### 테스트 번호 사용 시:
- ✅ 실제 SMS가 전송되지 않습니다
- ✅ Firebase Console에 설정한 인증 코드를 사용합니다
- ✅ 개발 중에는 SHA-1 지문 없이도 작동할 수 있습니다
- ⚠️ 때로는 "너무 많은 요청" 오류가 발생할 수 있습니다

### "너무 많은 요청" 오류 발생 시:
1. **5-10분 대기** 후 다시 시도
2. Firebase Console에서 테스트 번호 재등록
3. 앱 재시작 후 다시 시도

## SHA-1 지문은 언제 등록하나요?

- ✅ **개발 중**: 테스트 번호만으로 충분 (지금은 이 단계)
- ✅ **프로덕션 배포 전**: SHA-1 지문 등록 필수
- ✅ **실제 SMS 사용 시**: SHA-1 지문 등록 필요

## 개발 단계 체크리스트

- [ ] Firebase Console에 테스트 번호 등록됨
- [ ] 전화번호 형식 확인 (`+821011112222`)
- [ ] 인증 코드 확인 (`111111`, `333333`)
- [ ] 앱에서 테스트 성공
- [ ] SHA-1 지문은 나중에 등록 예정

## 참고

- 테스트 번호는 개발 중에만 사용합니다
- 프로덕션 배포 시에는 실제 SMS 인증을 사용해야 합니다
- SHA-1 지문은 프로덕션 배포 전에 등록하면 됩니다
