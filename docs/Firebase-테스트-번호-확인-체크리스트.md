# Firebase Console 테스트 번호 확인 체크리스트

## 확인 절차

### 1단계: Firebase Console 접속

1. [Firebase Console](https://console.firebase.google.com/) 접속
2. 프로젝트 선택: **howareyou-1c5de**

### 2단계: Authentication 메뉴로 이동

1. 왼쪽 메뉴에서 **Authentication** 클릭
2. 상단 탭에서 **Sign-in method** 클릭
3. 목록에서 **전화번호** (Phone) 찾기
4. **전화번호** 클릭하여 설정 화면 열기

### 3단계: 전화번호 인증 활성화 확인

- **사용 설정** 토글이 **켜져 있는지** 확인 (파란색/ON 상태)
- 꺼져 있다면 토글을 켜고 저장

### 4단계: 테스트 전화번호 섹션 확인

**테스트 전화번호** 섹션으로 스크롤

#### 등록되어 있어야 할 번호:

##### ✅ 보호자 번호 (Pixel 1)
- **전화번호**: `+821011112222` 또는 `+82 10-1111-2222`
- **인증 코드**: `111111`
- **상태**: 등록되어 있어야 함

##### ✅ 보호 대상자 번호 (Pixel 2)
- **전화번호**: `+821033334444` 또는 `+82 10-3333-4444`
- **인증 코드**: `333333`
- **상태**: 등록되어 있어야 함

## 확인 체크리스트

### 필수 확인 사항:

- [ ] **전화번호 인증이 활성화**되어 있는가?
- [ ] **테스트 전화번호 섹션이 보이는가?**
- [ ] **보호자 번호 `+821011112222`가 등록**되어 있는가?
- [ ] **보호자 인증 코드가 `111111`**인가?
- [ ] **보호 대상자 번호 `+821033334444`가 등록**되어 있는가?
- [ ] **보호 대상자 인증 코드가 `333333`**인가?

## 문제 발견 시 해결 방법

### 테스트 번호가 없는 경우:

1. **테스트 전화번호 추가** 버튼 클릭
2. **전화번호 입력**: `+821011112222` (또는 `+82 10-1111-2222`)
3. **인증 코드 입력**: `111111`
4. **추가** 버튼 클릭
5. 두 번째 번호도 동일하게 추가:
   - 전화번호: `+821033334444`
   - 인증 코드: `333333`
6. **저장** 확인

### 테스트 번호가 잘못된 경우:

1. **기존 테스트 번호 삭제** (휴지통 아이콘 클릭)
2. 위의 절차대로 **새로 추가**
3. **저장** 확인

### 전화번호 형식이 다른 경우:

다음 형식 모두 가능합니다:
- `+821011112222` ✅
- `+82 10-1111-2222` ✅
- `+82-10-1111-2222` ✅

**중요**: 인증 코드는 정확히 일치해야 합니다:
- 보호자: `111111` (6자리 숫자)
- 보호 대상자: `333333` (6자리 숫자)

## 확인 후 다음 단계

### 테스트 번호가 올바르게 등록되어 있다면:

1. **앱 재시작**:
   ```powershell
   flutter run -d emulator-5554
   ```

2. **다시 시도**:
   - Pixel 1: `010-1111-2222` 입력 → 인증 코드 `111111`
   - Pixel 2: `010-3333-4444` 입력 → 인증 코드 `333333`

### 여전히 "너무 많은 요청" 오류가 발생한다면:

1. **5-10분 대기** 후 다시 시도
2. Firebase Console에서 프로젝트 상태 확인
3. 필요시 **SHA-1 지문 등록** (나중에)

## 참고

- 테스트 전화번호를 사용하면 실제 SMS가 전송되지 않습니다
- 인증 코드는 Firebase Console에 설정한 값을 사용합니다
- 테스트 번호만으로도 개발 중에는 충분히 사용 가능합니다
- SHA-1 지문은 프로덕션 배포 전에 등록하면 됩니다
