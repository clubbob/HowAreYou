## 2. 기능 상세

### 2.1 보호대상자 무제한 등록

| 구분 | 무료 | 프리미엄 |
|------|------|----------|
| 등록 한도 | 2명 | 무제한 |
| 판별 | `guardianSubjectCount >= 2` 시 추가 차단 | `isPremium()` 통과 시 추가 허용 |

**프리미엄 판별 로직**

```
subscriptionStatus ∈ { 'active', 'premium' }
AND (subscriptionExpiry == null OR subscriptionExpiry > now)
→ isPremium = true
```

**적용 시점**

| 시점 | 함수 | 동작 |
|------|------|------|
| 보호자가 전화번호로 대상자 추가 | `addGuardianToSubject` | 무료 2명 초과 시 reject |
| 대상자가 전화번호로 보호자 추가 | `addGuardianToSubjectBySubjectInvite` | 무료 2명 초과 시 reject |
| 신규 가입 시 대기 초대 자동 연결 | `processPendingInvitesOnSignup` | 무료 2명 초과 시 해당 초대 스킵 |

---

### 2.2 스마트 이상 감지 (6h/9h/12h)

**단계별 무활동 기준**

| 단계 | 무활동 시간 | 알림 문구 |
|------|-------------|-----------|
| 1 | 6시간 | 6시간째 안부가 없습니다. |
| 2 | 9시간 | 9시간째 안부가 없습니다. |
| 3 | 12시간 | 12시간째 안부가 없습니다. 확인이 필요합니다. |

**발송 조건 (핵심)**

```
baseAt = lastMovementAt ?? lastResponseAt
if (now - baseAt >= 단계시간) → 발송 후보
```

- **원칙**: `lastMovementAt` (핸드폰 물리적 이동)
- **보조**: `lastMovementAt`이 없거나 센서 비정상 시에만 `lastResponseAt`(앱 기록) 사용
- **lastResponseAt 단독 기준은 사용하지 않음**

**안전장치**

| 구분 | 조건 | 목적 |
|------|------|------|
| **A** | `lastInactivityAlertLevel == 현재단계` AND `lastInactivityAlertAt` 3h 이내 | 같은 단계 재발송 방지 |
| **B** | `현재단계 <= lastInactivityAlertLevel` | 단계 역행 방지 (2단계 후 1단계로 떨어지는 것 방지) |

---

### 2.3 실시간 무활동 알림

| 항목 | 내용 |
|------|------|
| **스케줄** | 30분마다 (`*/30 * * * *`, Asia/Seoul) |
| **수신자** | 해당 보호대상자와 연결된 **프리미엄 보호자만** |
| **FCM data.type** | `INACTIVITY_ALERT` |
| **FCM data** | subjectId, subjectDisplayName, stage(1/2/3) |

## 3. 핸드폰 이동 감지 (lastMovementAt)

### 3.1 원칙

| 구분 | 설명 |
|------|------|
| **의미** | 보호대상자 핸드폰의 **물리적 이동** 시각 |
| **목적** | 앱 기록 없이도 “움직임이 없는지” 감지 |

### 3.2 갱신 경로

| 경로 | 시점 | 역할 |
|------|------|------|
| **MovementDetectionService** | WALKING, RUNNING 등 감지 시 | 원칙 — Activity Recognition |
| **onResponseCreated/Updated** | 안부 기록 시 | 보조 — 센서 미동작 시 대체 |

### 3.3 MovementDetectionService 상세

| 항목 | 내용 |
|------|------|
| **패키지** | `activity_recognition_flutter` |
| **호출** | `SubjectModeScreen` 진입 시 `start(subjectId)`, 종료 시 `stop()` |
| **활동 타입** | WALKING, RUNNING, ON_FOOT, IN_VEHICLE, ON_BICYCLE, TILTING (STILL 제외) |
| **최소 신뢰도** | 50% |
| **업데이트 간격** | 최소 5분 (중복 쓰기 방지) |
| **배경 동작** | `runForegroundService: true` (Android) |

### 3.4 권한

- Android: `ACTIVITY_RECOGNITION`, `com.google.android.gms.permission.ACTIVITY_RECOGNITION`
- `Permission.activityRecognition.request()` 호출

## 4. 데이터 구조

### 4.1 users

| 필드 | 타입 | 용도 |
|------|------|------|
| `subscriptionStatus` | string | `active` / `premium` → 프리미엄 |
| `subscriptionExpiry` | Timestamp | 구독 만료일 (null = 무기한) |
| `guardianSubjectCount` | number | 보호자별 등록 대상자 수 |

### 4.2 subjects

| 필드 | 타입 | 용도 |
|------|------|------|
| `lastMovementAt` | Timestamp \| null | 마지막 핸드폰 물리적 이동 시각 |
| `lastResponseAt` | Timestamp | 마지막 앱 안부 기록 시각 (기존) |
| `lastInactivityAlertAt` | Timestamp \| null | 마지막 무활동 알림 발송 시각 |
| `lastInactivityAlertLevel` | number | 마지막 발송 단계 (0=미발송, 1=6h, 2=9h, 3=12h) |

### 4.3 리셋 조건

**① 안부 기록 시 (onResponseCreated/onResponseUpdated)**

```
lastInactivityAlertAt = null (필드 삭제)
lastInactivityAlertLevel = 0
lastMovementAt = serverTimestamp()  // 보조 갱신
```

**② 움직임 감지 시 (onSubjectUpdated) — P0 치명 버그 방지**

```
lastMovementAt 갱신 감지 → lastInactivityAlertAt = null, lastInactivityAlertLevel = 0
```

- 12h 알림 후 움직임만 생기고 응답 없으면, (B) 단계 역행 방지 때문에 새 사이클 알림이 막히는 문제 해결
- "움직임이 생기면 새로운 사이클 시작"

## 5. Cloud Functions 목록

| 함수 | 스케줄/트리거 | 역할 |
|------|--------------|------|
| `addGuardianToSubject` | Callable | 보호자→대상자 추가 (프리미엄 체크) |
| `addGuardianToSubjectBySubjectInvite` | Callable | 대상자→보호자 추가 (프리미엄 체크) |
| `removeGuardianFromSubject` | Callable | 보호자 제거 (guardianSubjectCount -1) |
| `removeGuardianFromSubjectBySubject` | Callable | 대상자→보호자 제거 (guardianSubjectCount -1) |
| `processPendingInvitesOnSignup` | Auth onCreate | 신규 가입 시 대기 초대 처리 (프리미엄 체크) |
| `onResponseCreated` | Firestore onCreate | 기록 시 보호자 알림 + lastInactivityAlert* 리셋 + lastMovementAt 갱신 |
| `onResponseUpdated` | Firestore onUpdate | 기록 수정 시 보호자 알림 + lastInactivityAlert* 리셋 + lastMovementAt 갱신 |
| `onSubjectUpdated` | Firestore onUpdate | lastMovementAt 갱신 시 lastInactivityAlert* 리셋 (P0 치명 버그 방지) |
| `sendPremiumInactivityAlert` | Pub/Sub 30분마다 | 6h/9h/12h 무활동 알림 (프리미엄 보호자만) |

## 6. sendPremiumInactivityAlert 처리 흐름

```
1. lastResponseAt < 6h, lastMovementAt < 6h 문서 각각 쿼리 → 병합
2. 각 문서에 대해:
   - baseAt = lastMovementAt ?? lastResponseAt
   - inactiveMs = now - baseAt
   - currentStage = 1(6h) | 2(9h) | 3(12h)
   - (B) currentStage <= lastInactivityAlertLevel → 스킵
   - (A) 같은 단계 + 3h 이내 → 스킵
   - pairedGuardianUids 중 isPremium() 필터
   - premiumGuardians가 없으면 → 스킵
   - FCM 발송 (INACTIVITY_ALERT)
   - lastInactivityAlertAt, lastInactivityAlertLevel 업데이트
```
