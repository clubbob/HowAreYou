# 편리한 실행 방법

## 문제
`.\run-android.ps1` 같은 긴 명령어를 외워야 해서 불편함

## 해결 방법

### 방법 1: 별칭 설정 (가장 편리함) ⭐

#### 1단계: 별칭 설정 스크립트 실행

프로젝트 루트에서:

```powershell
.\setup-flutter-alias.ps1
```

#### 2단계: 현재 세션에 적용

```powershell
. $PROFILE
```

#### 3단계: 간단하게 사용!

```powershell
fr -d emulator-5554    # flutter run 대신 fr 사용 (EGL 로그 자동 필터링)
fd                     # flutter devices 대신 fd 사용 (번호 선택)
frs                    # 에뮬레이터 선택 후 실행
```

**이제 `flutter run` 대신 `fr`만 입력하면 됩니다!**

### 방법 2: VSCode/Cursor 기본 빌드 사용 (권장)

#### Ctrl+Shift+B 사용

1. **Ctrl+Shift+B** 키 누르기
2. 자동으로 EGL 로그가 필터링된 상태로 실행됩니다
3. 별칭 설정 없이도 바로 사용 가능!

#### 또는 작업 메뉴에서

1. `Ctrl+Shift+P` → "Tasks: Run Task"
2. **"Flutter Run (EGL 로그 제외)"** 선택

### 방법 3: 프로젝트별 별칭 (프로젝트 내에서만 작동)

프로젝트 루트에 `.ps1` 파일을 만들어서:

```powershell
# fr.ps1 파일 생성
.\run-android.ps1 @args
```

그러면:
```powershell
.\fr -d emulator-5554
```

## 사용 가능한 별칭 (설정 후)

| 별칭 | 원래 명령어 | 설명 |
|------|------------|------|
| `fr` | `flutter run` | 앱 실행 (EGL 로그 자동 필터링) |
| `fd` | `flutter devices` | 기기 목록 (번호 선택) |
| `frs` | `flutter run` (선택) | 에뮬레이터 선택 후 실행 |

## 사용 예시

### 기본 실행
```powershell
fr -d emulator-5554
```

### 에뮬레이터 선택
```powershell
frs
```

### 기기 목록 확인
```powershell
fd
```

## 별칭 제거 방법

PowerShell 프로필에서 제거:

```powershell
notepad $PROFILE
```

파일에서 "Flutter 별칭" 섹션을 삭제하고 저장

## 권장 방법

**가장 편리한 순서:**

1. ⭐ **별칭 설정** (`.\setup-flutter-alias.ps1` 실행)
2. **`fr` 명령어 사용** (가장 간단!)
3. 또는 **Ctrl+Shift+B** 사용 (VSCode/Cursor)

## 참고

- 별칭은 PowerShell 프로필에 저장되므로 모든 PowerShell 세션에서 사용 가능합니다
- 프로젝트를 변경해도 별칭은 계속 작동합니다
- 별칭을 설정하지 않아도 `Ctrl+Shift+B`는 항상 사용 가능합니다
