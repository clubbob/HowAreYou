# 테스트 번호 확인 단계

## 현재 화면 상태

### ✅ 확인된 사항:
- **전화번호 인증**: "사용 설정" ON (활성화됨) ✅
- **테스트용 전화번호 섹션**: 화면에 보임 (확장 필요)

## 다음 단계: 테스트 번호 확인

### 1단계: "테스트용 전화번호(선택사항)" 섹션 확장

현재 화면에서 **"테스트용 전화번호(선택사항)"** 섹션을 클릭하여 펼치세요.

### 2단계: 등록된 테스트 번호 확인

섹션을 펼치면 다음이 보여야 합니다:

#### 등록되어 있어야 할 번호:

##### 보호자 번호 (Pixel 1)
- **전화번호**: `+821011112222` 또는 `+82 10-1111-2222`
- **인증 코드**: `111111`

##### 보호 대상자 번호 (Pixel 2)
- **전화번호**: `+821033334444` 또는 `+82 10-3333-4444`
- **인증 코드**: `333333`

## 확인 체크리스트

- [ ] "테스트용 전화번호(선택사항)" 섹션을 펼쳤는가?
- [ ] 보호자 번호 `+821011112222`가 등록되어 있는가?
- [ ] 보호자 인증 코드가 `111111`인가?
- [ ] 보호 대상자 번호 `+821033334444`가 등록되어 있는가?
- [ ] 보호 대상자 인증 코드가 `333333`인가?

## 테스트 번호가 없는 경우

### 추가 방법:

1. **"테스트용 전화번호(선택사항)"** 섹션 펼치기
2. **"전화번호 추가"** 또는 **"추가"** 버튼 클릭
3. **전화번호 입력**: `+821011112222` (또는 `+82 10-1111-2222`)
4. **인증 코드 입력**: `111111`
5. **추가** 또는 **저장** 버튼 클릭
6. 두 번째 번호도 동일하게 추가:
   - 전화번호: `+821033334444`
   - 인증 코드: `333333`
7. 하단의 **"저장"** 버튼 클릭

## 참고사항

### SHA-1 지문 안내:
- 화면에 "SHA-1 출시 디지털 지문"에 대한 안내가 보입니다
- **개발 단계에서는 테스트 번호만으로도 작동할 수 있습니다**
- SHA-1 지문은 나중에 프로덕션 배포 전에 등록하면 됩니다

### 일일 SMS 할당량:
- 화면 상단의 경고는 테스트 번호 사용 시 영향 없습니다
- 테스트 번호를 사용하면 실제 SMS가 전송되지 않습니다

## 다음 단계

테스트 번호를 확인/등록한 후:

1. **하단 "저장" 버튼 클릭** (변경사항이 있다면)
2. **앱 재시작**:
   ```powershell
   flutter run -d emulator-5554
   ```
3. **다시 시도**:
   - Pixel 1: `010-1111-2222` 입력 → 인증 코드 `111111`
   - Pixel 2: `010-3333-4444` 입력 → 인증 코드 `333333`
