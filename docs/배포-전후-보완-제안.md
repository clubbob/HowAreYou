# 앱 배포 전·후 보완 제안 (하나씩)

배포에 필수는 아니지만, 순서대로 하나씩 적용해 보시면 좋은 항목들입니다.

---

## 1. Release 서명 설정 (배포 필수)

**현재**: `android/app/build.gradle.kts`에서 release 빌드가 **debug 키**로 서명됨.  
**문제**: Play Store는 release용 전용 서명이 필요함.  
**제안**: 키스토어 생성 후 `signingConfigs`에 release 설정 추가하고, `buildTypes.release.signingConfig`에 연결. 비밀번호는 `key.properties`(git 제외)로 관리.

---

## 2. Application ID 변경 (권장)

**현재**: `com.example.how_are_you`  
**문제**: `com.example.*`는 샘플용이라 스토어/실서비스 앱에는 부적절.  
**제안**: 예) `com.회사명또는서비스명.howareyou` 등 고유한 패키지명으로 변경. 변경 시 Firebase 콘솔 앱 패키지명, `firebase_options.dart`(재생성) 동기화 필요.

---

## 3. 알림 탭 시 화면 이동 (UX)

**현재**: `lib/services/fcm_service.dart`에 TODO — "보호자 대시보드로 이동하거나 알림 상세 화면 표시"  
**제안**: FCM 알림 탭 시 로그인 상태면 보호자 대시보드(또는 해당 대상자 상세)로 바로 이동하도록 `FirebaseMessaging.onMessageOpenedApp` / `getInitialMessage` 처리 보완.

---

## 4. 미회신 시 대상자에게 알림 (기능)

**현재**: `lib/screens/no_response_screen.dart`에 TODO — "FCM으로 대상자에게 '기분 알려주기' 알림 전송"  
**제안**: 보호자가 "미회신" 화면에서 요청 시, 해당 보호대상자 기기로 FCM으로 "기분 알려주기" 유도 알림 전송. (Cloud Functions 또는 클라이언트에서 FCM 토큰으로 전송)

---

## 5. 에러/예외 메시지 사용자 친화화 ✅ 완료

**적용함**: `저장 실패: $e` 등 → "잠시 후 다시 시도해 주세요.", "네트워크를 확인해 주세요." 등으로 치환 완료.

---

## 6. 앱 이름·아이콘 확인

**현재**: `AndroidManifest`에 `android:label="지금 어때?"`, `android:icon="@mipmap/ic_launcher"`  
**제안**: 스토어에 노출할 앱 이름과 실제 기기 설치 후 이름이 일치하는지 확인. 필요 시 아이콘·스플래시를 브랜드에 맞게 교체.

---

## 7. 개인정보 처리 방침·이용약관 (스토어 필수)

**현재**: 별도 문서 미확인.  
**제안**: Play Store 출시 시 개인정보처리방침 URL 필수. 전화번호 수집·Firebase 사용·알림 목적 등을 담은 페이지 하나 준비 후 스토어 등록 정보에 URL 입력.

---

## 8. 프로젝트 문서 최신화

**현재**: `docs/프로젝트-구현-현황-최신.md` 등에 예전 스펙(하루 3회, "고마워요" 표시 등)이 남아 있을 수 있음.  
**제안**: 하루 1회 응답, 별로 입력창 제거, 고맙습니다 알림 제거 등 최근 변경 사항 반영해 문서 한 번 정리.

---

## 9. 릴리스 빌드에서 디버그 로그 최소화

**현재**: `debugPrint` 필터링은 디버그 모드에만 적용.  
**제안**: `kReleaseMode`일 때는 로그 출력 자체를 줄이거나, 민감 정보가 포함되지 않도록 메시지 내용 검토.

---

## 10. 버전 코드·버전 이름 정책

**현재**: `pubspec.yaml`에 `version: 1.0.0+1`  
**제안**: 스토어 업데이트 시마다 버전 코드(+1) 증가, 버전 이름(1.0.0)은 정책에 맞게 올리는 규칙을 정해 두고, 필요하면 CI/스크립트로 자동 증가.

---
