# 별칭 설정 문제 해결

## 문제
`fr` 명령어가 작동하지 않음

## 해결 방법

### 1단계: 별칭 설정 스크립트 실행

```powershell
.\setup-flutter-alias-simple.ps1
```

### 2단계: 현재 세션에 적용 (필수!)

별칭을 설정한 후, **현재 PowerShell 세션에 적용**해야 합니다:

```powershell
. $PROFILE
```

이 명령어를 실행하지 않으면 별칭이 작동하지 않습니다!

### 3단계: 별칭 확인

```powershell
Get-Command fr
```

명령어가 보이면 성공입니다.

## 사용 방법

별칭이 설정되면:

```powershell
fr -d emulator-5554    # flutter run 대신 fr 사용
fd                     # flutter devices 대신 fd 사용
frs                    # 에뮬레이터 선택 후 실행
```

## 문제 해결

### "fr: 명령을 찾을 수 없습니다" 오류

1. **별칭 설정 확인**:
   ```powershell
   .\setup-flutter-alias-simple.ps1
   ```

2. **현재 세션에 적용**:
   ```powershell
   . $PROFILE
   ```

3. **프로필 파일 확인**:
   ```powershell
   notepad $PROFILE
   ```
   파일에 "Flutter Run 별칭" 섹션이 있는지 확인

### 여전히 안 되는 경우

#### 방법 1: 프로필 파일 직접 확인
```powershell
notepad $PROFILE
```

파일 끝에 다음이 있는지 확인:
```powershell
# Flutter Run 별칭 (EGL 로그 자동 필터링) - HowAreYou 프로젝트
function fr {
    ...
}
```

#### 방법 2: 프로필 파일 수동 로드
```powershell
. $PROFILE
```

#### 방법 3: 새 PowerShell 창 열기
- 현재 PowerShell 창 닫기
- 새 PowerShell 창 열기
- 별칭이 자동으로 로드됨

## 빠른 해결

터미널에서 다음 명령어를 순서대로 실행:

```powershell
# 1. 별칭 설정
.\setup-flutter-alias-simple.ps1

# 2. 현재 세션에 적용 (중요!)
. $PROFILE

# 3. 확인
Get-Command fr

# 4. 사용
fr -d emulator-5554
```

## 참고

- 별칭은 PowerShell 프로필에 저장됩니다
- 새 PowerShell 세션을 열면 자동으로 로드됩니다
- 현재 세션에서는 `. $PROFILE` 명령어로 수동 로드해야 합니다
