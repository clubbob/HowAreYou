# 지금 어때 홈페이지 (Landing MVP) 구축 가이드

## 완료된 작업

- `homepage/` 폴더에 Next.js 15 + TypeScript + Tailwind CSS 프로젝트 생성
- 메인 랜딩 페이지 (Hero, 문제 제기, 서비스 방식, 신뢰, 공지사항, CTA, Footer)
- 베타 테스트 참여 모달 (이메일 수집 → Firestore `waitlist`)
- 공지사항 섹션 (Firestore `announcements` 연동)
- 개인정보처리방침 페이지 (`/privacy`) - 필수 7개 항목 포함
- Firebase 보안 규칙 추가 (`waitlist`, `announcements`)
- 로고 (`assets/images/logo.png` → `homepage/public/logo.png`)
- SEO 메타 태그, og:image

```bash
cd homepage
npm install   # 최초 1회
npm run dev
```

## Firebase 설정 (필수)

1. [Firebase Console](https://console.firebase.google.com/) → 프로젝트 **howareyou-1c5de** 선택
2. 프로젝트 설정 → 일반 → 내 앱 → **웹 앱 추가** (이미 있으면 사용)
3. `homepage/.env.local` 생성 후 config 값 입력:

```
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=howareyou-1c5de.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=howareyou-1c5de
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=howareyou-1c5de.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NEXT_PUBLIC_FIREBASE_APP_ID=...
```

4. Firestore 보안 규칙 배포:

```bash
firebase deploy --only firestore:rules
```

## Firestore 컬렉션 구조

### waitlist (베타 신청)

| 필드 | 타입 | 설명 |
|------|------|------|
| email | string | 이메일 |
| createdAt | timestamp | 등록 시각 |

### announcements (공지사항)

| 필드 | 타입 | 설명 |
|------|------|------|
| title | string | 제목 |
| content | string | 내용 |
| createdAt | timestamp | 작성일 |
| pinned | boolean | (선택) 상단 고정 |

Firebase Console에서 직접 문서 추가 가능. 관리자만 쓰기 허용됨.

## Vercel 배포

1. Vercel 대시보드 → 프로젝트 추가 → GitHub 연결
2. **Root Directory를 반드시 `homepage`로 지정** ← 중요!
   - 프로젝트 설정 → General → Root Directory → `homepage` 입력
   - 지정하지 않으면 레포 루트(Flutter)가 빌드되어 `firebase_options.dart` 오류 발생
3. 환경 변수에 `NEXT_PUBLIC_FIREBASE_*` 추가
4. 배포
