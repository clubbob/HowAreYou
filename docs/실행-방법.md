# 앱 실행 방법 요약

## EGL 로그 반복을 피하려면 (권장)

**`flutter run` 직접 실행 시 EGL 로그가 반복됩니다.** 아래만 사용하세요.

| 방법 | 명령/동작 | 비고 |
|------|-----------|------|
| Cursor 빌드 | **Ctrl+Shift+B** | EGL 로그 제외 (기본 빌드 작업) |
| 터미널 (PowerShell) | `.\run-android.ps1` | EGL 로그 제외 |
| 터미널 (CMD 등) | `.\run.bat` | EGL 로그 제외 (내부에서 run-android.ps1 호출) |
| Cursor F5 | **실행** → **디버깅 시작** (F5) | 디버그 콘솔 필터에 `!EGL_emulation` 입력 시 로그 숨김 |

## 일반 실행 (로그 그대로 출력됨)

| 방법 | 명령/동작 |
|------|-----------|
| 터미널 | `flutter run` |

기기 지정: `.\run-android.ps1 -d emulator-5554` 또는 `.\run.bat -d emulator-5554`

자세한 내용은 `ANDROID_EMULATOR_GUIDE.md`의 "터미널에 D/EGL_emulation app_time_stats 로그가 반복될 때" 섹션 참고.
