# Cursor 터미널에서 에뮬레이터 변경 방법

## 빠른 답변

### 방법 1: 에뮬레이터 선택 스크립트 사용 (가장 간단) ⭐

```powershell
.\run-select-device.ps1
```

또는 배치 파일:

```cmd
run-select.bat
```

실행하면:
1. 사용 가능한 에뮬레이터 목록이 번호와 함께 표시됩니다
2. 원하는 번호를 입력하면 해당 에뮬레이터에서 앱이 실행됩니다

**예시:**
```
사용 가능한 에뮬레이터:

  [1] Pixel 6 (보호자) - emulator-5554
  [2] Pixel 7 (보호 대상자) - emulator-5556

사용할 에뮬레이터 번호를 선택하세요 (1-2): 1
```

---

### 방법 2: 직접 기기 ID 지정

```powershell
flutter run -d emulator-5554    # Pixel 6
flutter run -d emulator-5556    # Pixel 7
```

---

### 방법 3: 별칭 사용 (설정 후)

별칭을 설정했다면:

```powershell
fr -d emulator-5554    # fr = flutter run (EGL 로그 필터링 포함)
frs                     # 에뮬레이터 선택 후 실행
```

별칭 설정 방법:
```powershell
.\setup-flutter-alias.ps1
. $PROFILE
```

---

## 현재 실행 중인 에뮬레이터 확인

```powershell
flutter devices
```

또는 번호로 선택:

```powershell
.\flutter-devices.ps1
```

---

## 이미 실행 중인 앱을 다른 에뮬레이터로 변경

### 방법 1: 앱 종료 후 다시 실행

1. 현재 실행 중인 앱에서 **`q`** 키를 눌러 종료
2. 원하는 에뮬레이터로 다시 실행:
   ```powershell
   .\run-select-device.ps1
   ```

### 방법 2: Hot Restart 불가 (에뮬레이터 변경은 앱 재시작 필요)

에뮬레이터 변경은 앱을 완전히 종료하고 다시 실행해야 합니다.
Hot reload(`r`)나 Hot restart(`R`)로는 에뮬레이터를 변경할 수 없습니다.

---

## 자주 사용하는 명령어

| 명령어 | 설명 |
|--------|------|
| `.\run-select-device.ps1` | 에뮬레이터 선택 후 실행 (EGL 로그 필터링) |
| `flutter devices` | 연결된 기기 목록 확인 |
| `flutter run -d emulator-5554` | 특정 에뮬레이터에서 실행 |
| `.\flutter-devices.ps1` | 기기 목록을 번호로 표시 |

---

## 추천 방법

**가장 간단한 방법:**

```powershell
.\run-select-device.ps1
```

이 명령어 하나면:
- ✅ 에뮬레이터 목록 자동 표시
- ✅ 번호로 선택
- ✅ EGL 로그 자동 필터링
- ✅ 앱 자동 실행

---

## 문제 해결

### 에뮬레이터가 목록에 안 보일 때

1. Android Studio에서 에뮬레이터가 실행 중인지 확인
2. `flutter devices` 명령어로 확인
3. 에뮬레이터 재시작

### PowerShell 실행 정책 오류

```powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### 배치 파일이 안 될 때

PowerShell에서 직접 실행:
```powershell
powershell -ExecutionPolicy Bypass -File .\run-select-device.ps1
```
